# 🕵️ Spy Bot — Telegram-бот для игры «Шпион»

MVP-версия бота для игры в «Шпиона» (Spyfall) с 4 режимами и 3 форматами игры.

## Быстрый старт

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# впиши BOT_TOKEN (получить у @BotFather)

python bot.py
```

Бот запустится в режиме long polling — отдельный сервер/домен не нужен.

## Как играть

1. `/start` → «Создать комнату» → бот пошагово спрашивает:
   количество игроков → количество шпионов (с рекомендацией по правилам,
   либо «случайно») → режим игры → категория → формат игры.
2. После подтверждения бот выдаёт уникальный код комнаты (например, `AB12CD`).
3. Остальные игроки жмут «Присоединиться по коду» и вводят код
   (или сразу открывают `t.me/ваш_бот?start=AB12CD`).
4. Когда набралось нужное количество игроков — у хоста появляется
   кнопка «🚀 Начать игру». Каждому в личку прилетает картинка + слово
   (или роль, или карточка шпиона).
5. Дальше — в зависимости от формата:
   - **Офлайн** — обсуждаете вслух, шпиона вычисляете сами. Хост может
     завершить раунд командой `/reveal`.
   - **Онлайн по очереди** — `/next` объявляет, чей ход (используйте в общем чате).
   - **Онлайн с голосованием** — обсуждаете где угодно (Discord и т.п.),
     а когда готовы — хост пишет `/vote`, и каждый анонимно голосует прямо
     в чате с ботом. Бот сам считает голоса и объявляет победителя.
6. Шпион в любой момент может нажать «🤫 Я знаю слово!» и написать вариант —
   игра завершится сразу.

## Архитектура

```
bot.py                     — точка входа, регистрация роутеров aiogram
app/
  config.py                 — переменные окружения, правила по числу шпионов
  models.py                 — Room, Player, enums (режимы/форматы/статусы)
  storage.py                — in-memory хранилище комнат (см. "Известные ограничения")
  game_data.py              — словари категорий/локаций-ролей
  game_logic.py              — распределение слов/ролей/шпионов
  images.py                  — картинки: локальные файлы из assets/images + фолбэк на Pillow
  keyboards.py                — все инлайн-клавиатуры
  states.py                    — FSM-состояния визарда создания комнаты
  handlers/
    start.py                    — /start, вход по deep-link с кодом комнаты
    create_room.py               — визард создания комнаты
    join_room.py                  — присоединение по коду
    lobby.py                       — отображение состава комнаты, выход
    game.py                         — старт игры, раздача карточек, /next, /reveal, догадка шпиона
    voting.py                        — /vote и анонимное голосование
```

Игровая логика (`game_logic.py`) и модели (`models.py`) не зависят от Telegram —
их легко покрыть тестами или переиспользовать в другом интерфейсе.

## Реализованные режимы

| Режим | Что происходит |
|---|---|
| 📝 Классика | Все получают одно слово, шпион(ы) — ничего |
| 🎭 Роли | Все получают уникальную роль в общей локации, шпион(ы) — ничего |
| 🕵️ Двойной агент | Почти все получают слово А, «агент(ы)» — похожее слово Б |
| 👑 Ведущий | Ведущий сам вводит слово текстом и не играет |

## Допущения, которые я принял за тебя (легко поменять)

Раз мы сразу шли в код, вот решения, которые пришлось зафиксировать —
если что-то не подходит, скажи, поправим:

1. **Хост считается игроком** во всех режимах, кроме «Ведущего» — то есть
   он тоже может получить слово/роль и даже стать шпионом. В режиме
   «Ведущий» хост не играет и не входит в счётчик «количество игроков».
2. **Голосование при равенстве голосов** (или если большинство выбрало не
   шпиона) → победа шпиона. Правило меняется в одном месте —
   `_maybe_finish_voting` в `app/handlers/voting.py`.
3. **Формат «онлайн по очереди»** реализован в упрощённом виде: бот не
   ограничивает, кто может писать в чате (Telegram не позволяет боту
   «выключать микрофон» игрокам без прав администратора и явного мьюта) —
   он просто последовательно объявляет, чей сейчас ход, командой `/next`.
   Голосовые сообщения пока не передаются через бота — команда `/next`
   служит модератором очереди, а сам разговор происходит вживую/в звонке.
4. **Хранение состояния — в памяти процесса** (обычные Python-словари).
   Это самый простой вариант для MVP, но при перезапуске бота все текущие
   комнаты потеряются, и на нескольких инстансах бот не будет работать
   корректно. Для продакшена — заменить `app/storage.py` на Redis/Postgres,
   интерфейс класса `Storage` можно оставить тем же.
5. **Картинки** — берутся из локальной папки `app/assets/images/` по имени
   файла, которое соответствует слову/роли (см. `IMAGES_NEEDED.md` — там
   точный список всех нужных файлов и правило именования). Если для
   конкретного слова файла не нашлось — бот сам рисует простую текстовую
   карточку через Pillow, чтобы игра не ломалась из-за нехватки картинок.
   Можно начать с одной категории и постепенно добавлять остальные.
6. **Категории и списки слов** — я заполнил по 12–15 примеров на категорию
   (Dota 2, LoL, фильмы, актёры, животные) и 5 локаций с ролями (зоопарк,
   больница, ресторан, самолёт, школа). Этого достаточно для теста, но для
   реальной игры стоит расширить списки — просто дополни словари в
   `app/game_data.py`.
7. **Режим «Ведущий»** — ведущий не видит подсказок от бота (кто шпион он
   и так знает, т.к. сам всё задумал), голосование ему доступно так же,
   как и обычному хосту (`/vote`, `/reveal`).

## Что стоит добавить дальше (не реализовано в MVP)

- Таймер обсуждения с авто-уведомлением («осталось 2 минуты»).
- Привязка комнаты к конкретному группового чату для `/next` и передача
  голосовых сообщений через бота по очереди (технически реализуемо —
  бот может принимать `voice` в личке у игрока и форвардить в группу).
- Персистентное хранилище (Redis/Postgres) вместо памяти процесса.
- Возможность кикать игроков из лобби, менять состав после старта.
- Более гибкая настройка правила ничьей при голосовании (сейчас победа
  шпиона).
- Локализация на другие языки (сейчас только русский).

## Деплой на Ubuntu-сервер (автозапуск через systemd)

Бот работает через long polling — сам подключается к Telegram, входящий
порт открывать не нужно, достаточно исходящего доступа в интернет с сервера.

### 1. Куда закинуть бота

Стандартное место для самостоятельно установленного софта на Ubuntu — `/opt`.
Рекомендую:

```
/opt/spybot/
```

(если проще — можно и в домашнюю папку, например `/home/твой_юзер/spybot`,
для личного сервера это тоже нормально; дальше просто подставляй свой путь
везде вместо `/opt/spybot`).

### 2. Перенос файлов на сервер

Со своего компьютера, где лежит скачанный `spy_bot.zip`:

```bash
scp spy_bot.zip твой_юзер@IP_СЕРВЕРА:/tmp/
```

Дальше всё — уже на сервере, по SSH (`ssh твой_юзер@IP_СЕРВЕРА`):

```bash
cd /tmp
unzip spy_bot.zip
sudo mv spy_bot /opt/spybot
sudo chown -R $USER:$USER /opt/spybot
cd /opt/spybot
```

### 3. Установка Python-зависимостей

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip

python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 4. Настройка токена

```bash
cp .env.example .env
nano .env
```

Вписываешь `BOT_TOKEN=...` (получить у [@BotFather](https://t.me/BotFather)
в Telegram). Сохраняешь (`Ctrl+O`, Enter,
`Ctrl+X` в nano).

### 5. Пробный запуск (проверить, что всё работает)

```bash
python bot.py
```

Если видишь в консоли лог без ошибок и бот отвечает на `/start` в
Telegram — всё в порядке. Останавливаешь пробный запуск через `Ctrl+C`.

### 6. Автозапуск через systemd

Создаёшь юнит-файл:

```bash
sudo nano /etc/systemd/system/spybot.service
```

Вставляешь (замени `твой_юзер` на своё имя пользователя — узнать можно
командой `whoami`):

```ini
[Unit]
Description=Spy Bot Telegram
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=твой_юзер
WorkingDirectory=/opt/spybot
ExecStart=/opt/spybot/venv/bin/python /opt/spybot/bot.py
EnvironmentFile=/opt/spybot/.env
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Сохраняешь и запускаешь:

```bash
sudo systemctl daemon-reload
sudo systemctl enable spybot     # автозапуск при загрузке сервера
sudo systemctl start spybot
sudo systemctl status spybot     # должно быть "active (running)"
```

### 7. Логи и управление

```bash
journalctl -u spybot -f          # логи в реальном времени (Ctrl+C — выйти)
sudo systemctl restart spybot    # перезапустить (например, после git pull / обновления кода)
sudo systemctl stop spybot       # остановить
```

Если после `systemctl start` статус не `active (running)` — почти всегда
причина видна прямо в `journalctl -u spybot -f`: неверный путь в
`ExecStart`, не тот `User`, не активирован venv в пути, или не задан
`BOT_TOKEN` в `.env`.

## Известные ограничения

- Число ролей в локациях (режим «Роли») ограничено 12 — при большем
  количестве игроков часть получит роль «Гость». Просто добавь роли в
  `app/game_data.py`, если нужно больше 12 играющих одновременно.
- Бот не проверяет права администратора в группах — команды `/next`,
  `/vote`, `/reveal` работают в любом чате, где написаны участником комнаты.
